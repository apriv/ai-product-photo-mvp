# 海报生成功能设计文档

> 状态：v1 设计中。本文档用于明确第一版交付边界，并保留 v2+ 演进方向。

## 1. 背景

当前 `/image` 已支持上传商品图，并通过 fal.ai 生成不同风格的商品图。其中“社媒海报”依赖模型直接生成背景、构图和少量广告文字。

下一阶段要把海报生成拆成两层：

- **Layer 1：AI 生成产品图 + 背景**。模型只负责商品主体、背景、光影、氛围，不负责最终可控文案。
- **Layer 2：文字层**。用户直接在图片上的悬浮输入框里编辑标题、副标题、CTA，确认后渲染成最终海报。

这样能先解决 AI 文字不可控、拼写错误、改字需要重生成的问题，同时不引入过重的排版系统。

## 2. v1 目标

v1 只做一个稳定可用的海报编辑闭环：

1. 用户上传商品图。
2. 选择“社媒海报”生成 Layer 1 底图。
3. 生成成功后进入文字编辑区。
4. 用户从 4 个固定文字模板中选择一个。
5. 用户编辑标题、副标题、CTA。
6. 用户确认内容。
7. 系统渲染最终图片，并提供和当前生成结果一致的打开/下载入口。

v1 不追求智能排版，也不处理复杂避让，只提供“固定字体 + 固定位置 + 可编辑内容”的确定性结果。

## 3. 非目标

v1 暂不做：

- 多尺寸自由画布，仅先做 1:1 社媒海报。
- 自定义拖拽文字位置。
- 自定义字体上传。
- 自动推荐文案。
- 自动识别商品区域并避让。
- 智能 typography、动态布局。
- 多格式、多尺寸、导出历史等复杂导出配置。
- 服务端存储用户海报工程文件。
- 文字模板 Marketplace 或后台配置。

## 4. 用户体验

### 4.1 主流程

```
上传商品图
  -> 选择“社媒海报”
  -> 生成 Layer 1 底图
  -> 选择文字模板
  -> 直接在图片上编辑标题 / 副标题 / CTA
  -> 确认内容
  -> 渲染最终图
  -> 打开/下载图片
```

### 4.2 页面结构

在现有 `/image` 页面内演进，不新增独立路由：

- 上传区：沿用当前商品图片上传与压缩逻辑。
- 模板区：沿用当前图片模板选择。
- 结果区：生成成功后显示 AI 底图。
- 海报文字区：仅当选择“社媒海报”且生成成功后出现，文字输入框直接悬浮在图片对应位置上。
- 确认按钮：用户确认文字内容后，渲染最终图片。
- 保存入口：渲染完成后沿用当前“打开/下载图片”的体验。

### 4.3 文案输入

默认给用户三个字段：

| 字段 | 默认值 | 限制 |
|---|---|---|
| 标题 | `New Arrival` | 建议 4-32 个英文字符或 2-14 个中文字符 |
| 副标题 | `Premium product photography` | 建议 8-72 个英文字符或 4-28 个中文字符 |
| CTA | `Shop Now` | 建议 2-16 个英文字符或 2-8 个中文字符 |

v1 不单独放表单输入区，三个字段就是图片上的悬浮输入框。用户点到文字区域即可编辑，输入内容实时显示在图片上。

字号根据内容量自动调整：

- 文案短时使用模板预设字号。
- 文案变长时在模板允许范围内缩小字号。
- 超过单行宽度时优先换行，仍放不下再继续缩小。
- 到达最小字号仍放不下时，输入框提示内容过长，要求用户精简。

## 5. v1 模板设计

4 个固定模板都以 1:1 画布为基准，例如 1024 x 1024。为了适配不同生成图实际尺寸，模板位置使用百分比坐标，渲染时换算成像素。

### 5.1 模板 A：简洁

适合商品主体在中下部、背景较干净的图。

- 标题：顶部 10%，居中，大号粗体。
- 副标题：标题下方 17%，居中，小号常规。
- CTA：底部 84%，居中，胶囊按钮样式。
- 风格：留白多、信息克制、适合多数日常商品。

### 5.2 模板 B：奢华

适合商品主体偏右或中间偏右的图。

- 标题：左侧 8%，下方 66%，左对齐。
- 副标题：标题下方，左对齐。
- CTA：左侧 8%，下方 84%。
- 风格：杂志感、精致、适合香水/美妆/饰品/高客单价商品。

### 5.3 模板 C：科技感

适合商品主体偏左或底部稳定的图。

- 标题：右侧 8%，顶部 12%，右对齐。
- 副标题：标题下方，右对齐。
- CTA：右侧 8%，顶部 31%。
- 风格：干净、利落、偏未来感，适合数码/小家电/工具类商品。

### 5.4 模板 D：网感

适合主体占画面中心、背景复杂但底部可压暗的图。

- 底部增加半透明暗色渐变。
- 标题：底部 70%，居中。
- 副标题：底部 77%，居中。
- CTA：底部 87%，居中。
- 风格：社媒促销感强、信息醒目，适合风格海报。

## 6. 技术方案

### 6.1 分层模型

v1 输出由两部分组成：

- `backgroundImageUrl`：当前 `/api/image/generate` 返回的 AI 图片 URL。
- `posterTextLayer`：前端状态，包含模板 id 和三个文案字段。

示例：

```ts
type PosterTextLayer = {
  templateId: "minimal" | "luxury" | "tech" | "social";
  title: string;
  subtitle: string;
  cta: string;
};
```

### 6.2 模板配置

新增 `src/features/poster/text-templates.ts`，作为文字模板单一数据源。模板配置包括：

- 名称与描述。
- 标题、副标题、CTA 的位置。
- 字体族、字号比例、字重。
- 颜色、阴影、背景装饰。
- 最大宽度、对齐方式。

v1 字体优先使用系统字体，避免额外加载，并让浏览器预览和最终导出尽量一致。推荐：

- `Inter, Arial, sans-serif`
- `Georgia, serif`
- `Arial Black, Arial, sans-serif`
- `system-ui, sans-serif`

### 6.3 悬浮编辑

v1 使用 HTML/CSS 在图片上叠加可编辑输入框：

```tsx
<div className="poster-editor">
  <img src={generatedImage} alt="generated poster background" />
  <PosterInlineTextInput field="title" layer={posterTextLayer} />
  <PosterInlineTextInput field="subtitle" layer={posterTextLayer} />
  <PosterInlineTextInput field="cta" layer={posterTextLayer} />
</div>
```

编辑态和预览态是同一个区域，不再让用户在图片外填表。输入框默认无明显边框，聚焦时显示轻量描边，避免破坏海报观感。

编辑区域保持固定宽高比 `aspect-square`，移动端宽度自适应。

### 6.4 字号自适应

字号自适应由前端负责，用固定规则保证可预测：

- 每个文字字段在模板中配置 `maxFontSize`、`minFontSize`、`maxLines`、`maxWidth`。
- 用户输入后，前端根据字符数、容器宽度和行数计算字号档位。
- 标题优先保持视觉层级，副标题和 CTA 可更早缩小。
- CTA 保持按钮样式，文字过长时提示精简，不无限拉宽按钮。

### 6.5 确认与渲染

用户点击“确认并生成海报”后，系统直接导出当前浏览器里的海报预览 DOM，生成一张最终 PNG。这样最终图尽量等同于用户正在看的预览，包括字体、阴影、按钮大小和位置。

v1 不再让服务端重新绘制文字层，避免 HTML/CSS 预览和服务端 SVG/sharp 渲染不一致。

- 前端克隆海报预览节点。
- 将输入框转换成普通文字节点。
- 将计算后的 CSS 内联到克隆节点。
- 用 SVG `foreignObject` + Canvas 生成 PNG data URL。
- 返回给页面作为最终图片。

为避免跨域图片导致导出失败，生成接口会把 fal 返回图片转成 data URL 后再返回前端。“添加标题”模板本身也直接返回上传后的压缩图 data URL。

### 6.6 服务端变化

v1 不需要改变扣费模型：

- AI 生成 Layer 1 时按现有“社媒海报”价格扣费。
- 编辑文字、确认渲染和保存图片不额外扣费。
- `GenerationLog` 仍只记录 AI 生成，不记录每次保存。

可选增加轻量日志：

- 仅前端 console 或后续产品埋点记录所选文字模板。
- 暂不写数据库，避免扩大 v1 范围。

不新增保存接口。最终 PNG 在浏览器端生成，并以 data URL 展示和下载。

### 6.7 Prompt 调整

“社媒海报”的 prompt 需要去掉“让模型生成广告文字”的要求，改为：

- 保持商品真实不变。
- 生成适合承载文字的商业背景。
- 背景铺满整张图，商品不要占满画面，留出自然构图区域承载文字。
- 不要生成任何文字、logo、水印或乱码。

这样 Layer 1 更适合作为可控海报底图。

## 7. 文件改动建议

预计 v1 涉及：

| 文件 | 变更 |
|---|---|
| `src/features/image/templates.ts` | 调整“社媒海报” prompt，要求不生成文字，背景铺满但商品不要占满画面 |
| `src/features/poster/text-templates.ts` | 新增 4 个文字模板配置 |

如果实现时发现 `ImageGenerator.tsx` 过大，可以拆出：

- `PosterComposer.tsx`
- `PosterInlineTextInput.tsx`
- `render-poster.ts`

## 8. 验收标准

v1 完成时应满足：

- 上传商品图后仍可正常生成“社媒海报”。
- 生成后的底图不包含明显 AI 文字、乱码、水印。
- 用户可以在 4 个固定文字模板之间切换。
- 用户可以直接在图片上编辑标题、副标题、CTA。
- 输入内容实时显示在图片上。
- 字号能根据内容量自动缩小或换行，文字不溢出模板区域。
- 点击确认后能得到合成后的最终图片。
- 最终图片提供和当前生成结果一致的打开/下载入口。
- 移动端和桌面端文字控件不溢出、不遮挡主要操作。
- 余额、扣费、失败退款、生成日志保持现有行为。

## 9. 风险与取舍

### 9.1 AI 底图仍可能把商品放在文字区域

v1 用 prompt 留白和 4 个模板降低概率，不做自动避让。“网感”模板的底部暗色区域作为兜底。

### 9.2 浏览器导出兼容性

最终图由浏览器导出当前 DOM，预览一致性高于服务端重绘。但不同浏览器对 SVG `foreignObject` 的支持仍可能有差异，优先保证主流 Chromium 浏览器。

### 9.3 生成图转 data URL 失败

AI 生成成功后，服务端需要读取 fal 返回图片并转成 data URL。若 fal URL 下载失败，提示用户重新生成。

### 9.4 中文和英文排版差异

v1 只做通用换行和字号缩放。模板文案默认偏英文，中文可用但不做精细 typography。

## 10. v2+ 路线图

后续功能按价值和复杂度递进：

1. **多模板**：更多行业模板、节日模板、尺寸模板（1:1、4:5、9:16、16:9）。
2. **自动推荐文案**：根据商品品类、目标平台、促销目的生成标题、副标题、CTA。
3. **智能 typography**：根据文案长度自动选择字体、字号、行距、字重和对比度。
4. **动态布局**：根据图片构图和商品位置选择最佳文字区域。
5. **自动避让商品**：识别商品主体 mask 或 bounding box，避免文字覆盖商品。
6. **高级导出**：多格式、多尺寸、服务端存储和导出历史。
7. **品牌套件**：保存品牌色、字体、常用 CTA、口吻。
8. **工程文件**：保存可再次编辑的海报项目。
9. **批量生成**：一张商品图批量套多模板、多文案导出。

## 11. 推荐实施顺序

1. 调整社媒海报 prompt，确保 Layer 1 无文字、有留白。
2. 建立 4 个文字模板配置。
3. 在生成结果图上实现悬浮文字输入框。
4. 实现字号自适应规则。
5. 实现“确认并生成海报”的浏览器预览导出。
6. 最终图片沿用当前“打开/下载图片”入口。
7. 做移动端和桌面端检查：分别在手机宽度和电脑宽度下确认页面布局、悬浮输入框、字号自适应、确认渲染和打开/下载入口都正常。
